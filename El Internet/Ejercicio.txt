1. Pasos a apaso de lo que sucede desde cuendo el usuario escribre www.youtube.com desde un navegador web hasta que el video se reproduce en la pantalla.

Respuesta:
- El usuario escribe "www.youtube.com" en la barra de direcciones del navegador y presiona Enter.
- El navegador envia una solicitud al DNS (Servidor de Nombres de Dominio) para resolver el nombre de dominio "www.youtube.com" a una dirección IP.
- El DNS retorna la IP publica del servidor de YouTube
- El navegador envia una solicitud HTTP a la IP obtenida
- El servidor de YouTube recibe la solicitud y procesa la petición
- El servidor de YouTube envía los archivos HTML, CSS y JavaScript necesarios para cargar la página
- El navegador recibe los archivos y comienza a renderizar la página web
- El navegador ejecuta el código JavaScript para cargar los videos y demás elementos de la página
- El usuario selecciona un video y hace clic en él
- El navegador envía una solicitud al servidor de YouTube para obtener el video seleccionado
- El servidor de YouTube envía el archivo de video al navegador
- El navegador comienza a reproducir el video en la pantalla del usuario.

2. Frontend y Backend en acción
Suponga que está construyendo una app web para agendar citas médicas.
>Indique qué parte del sistema correspondería al frontend y cuál al backend.
>Mencione tres tecnologías posibles para cada uno.
>Explique brevemente cómo el frontend se comunicaría con el backend (mencione los conceptos de API, HTTP y request/response).

Respuesta:
- Frontend: La parte del sistema que el usuario ve e interactúa directamente, como la interfaz para agendar citas médicas.
- Tecnologías posibles: HTML/CSS, JavaScript, Bootstrap.
- Backend: La parte del sistema que maneja la lógica de negocio, la base de datos y la comunicación con el frontend.
- Tecnologías posibles: Python, Java, PHP.
- Comunicación entre frontend y backend: El frontend se comunicaría con el backend a través de una API (Interfaz de Programación de Aplicaciones) utilizando el protocolo HTTP. Cuando el usuario realiza una acción en la interfaz (por ejemplo, agendar una cita), el frontend envía una solicitud (request) HTTP al backend. El backend procesa esta solicitud, realiza las operaciones necesarias (como guardar la cita en la base de datos) y luego envía una respuesta (response) HTTP de vuelta al frontend, confirmando que la cita ha sido agendada exitosamente o proporcionando cualquier información adicional necesaria.
