Ordenes:
1.Lo primero es que no tenemos un Primary Key si no que varios numeros de orden que se repiten por cada item de la orden, por lo que se debe crear una tabal de detalle de ordenes (OrderItems) y una tabla principal de ordenes (Orders) donde se guarde un ID unico para cada orden (Order ID) y las columnas que tienen relacion directa con la orden. OrderItems debe tener su propio ID (Item ID) y una relacion con el Order ID de Orders. La tabla OrderItems debe contener la informacion de cada item Item ID, Item Name, Price, Quantity y Especial Request.
2. Luego en la columna Costumer Name se repriten varios Costumer Names con sus respectivos numeros de telefono (Costumer Phone), por lo que se debe crear una tabla aparte llamada Customers donde se guarden los nombres y los numeros de telefono, y un ID único para cada uno. Luego en la tabla Orders se reemplaza Costumer Names y Costumer Phone por el ID correspondiente (Costumer ID).
3. Veo que en la columna dirección se repiten varias direcciones; direcciones que tienen relacion directa con el costumer y relacion parcial con la orden, pero tambien hay un cliente con dos direcciones diferentes; eso me dice que cada costumer puede tener varias direcciones, por lo que se debe crear una tabla aparte llamada Addresses donde se guarden las direcciones y un ID único para cada una. Cada registro en esta table debe tener una relacion con el Id de Customers. Luego en la tabla original se reemplaza la dirección por el ID correspondiente (Adrees ID).
4. En la tabla OrderItems se repiten Items con sus respectivos Prices, por lo que se debe crear una tabla aparte llamada Products donde se guarden los nombres de los items y los precios, y un ID único para cada uno. Luego en la tabla OrderItems se reemplaza Item Name y Price por el ID correspondiente (Product ID).
5. Revisando uno de los item que se repiten "Fries" veo que el precio es diferente en dos ordenes diferentes de la tabla original, veo que la diferencia es que una tiene un espcial request de "Extra Ketchup" y esto afecta el precio en 1. Por lo tranto deberiamos agregar un producto adicional en la tabla Products llamado "Extra Ketchup" con su respectivo precio y agregar un Item en la tabla OrderItems para este producto adicional y relacionado a la respectiva orden. y agregar una columna Total Price en la tabla Orders para reflejar el precio total de la orden. 
6. Finalmente, la tabla Orders debe quedar con las columnas Order ID, Customer ID, Address ID y Delivery Time. Las otras tablas creadas serian Customers (Customer ID, Costumer Name, Costumer Phone), Addresses (Address ID, Customer ID, Address), OrderItems (Item ID, Order ID, Product ID, Quantity, Special Request) y Products (Product ID, Item Name, Price).

Automobiles:
1. En la tabla Automobiles en la columna Marca se repiten varias marcas, por lo que se debe crear una tabla aparte llamada Brands donde se guarden las marcas y un ID único para cada una. Luego en la tabla Automobiles se reemplaza Marca por el ID correspondiente (Brand ID).
2. Una misma Marca tiene varios Modelos y ademas el modelo esta mas relacionado a la Marca osea tiene una relacion parcial con el primary key, por lo que se debe crear una tabla aparte llamada Models donde se guarden los modelos y un ID único para cada uno. Cada registro en esta tabla debe tener una relacion con el Id de Brands. Luego en la tabla Automobiles se reemplaza Modelo por el ID correspondiente (Model ID).
3. La columna color tiene relacion directa con el primary key, pero se repiten varios colores, por lo que se debe crear una tabla aparte llamada Colors donde se guarden los colores y un ID único para cada uno. Luego en la tabla Automobiles se reemplaza Color por el ID correspondiente (Color ID).
4. La columna Owner Name y Owner Phone tienen relacion directa entre ellas y con el primary key, Pero se un owner puede tener varios automoviles, por lo que se debe crear una tabla aparte llamada Owners donde se guarden los nombres y los numeros de telefono, y un ID único para cada uno. Luego en la tabla Automobiles se reemplaza Owner Name y Owner Phone por el ID correspondiente (Owner ID).
5. Tambien tenemos las columnas Insurrance Company donde la compañias se pueden repetir, por lo que se debe crear una tabla aparte llamada Insurances donde se guarden las compañias y un ID único para cada una. Luego en la tabla Automobiles se reemplaza Insurrance Company por el ID correspondiente (Insurrance ID).
6.El VIN no puede ser el primary key porque se repite en varios registros, de hecho un mismo VIN puede tener varios dueños  por lo que tiene una relacion de muchos a muchos con los dueños, por lo que se debe crear una tabla intermedia llamada AutomobileOwners donde se guarde la relacion entre los automoviles y los dueños, con un ID unico (AutomobileOwner ID) y en la tabla Automobiles se elimina la columna Owner ID. y se agrega un ID unico (Automobile ID) como primary key y como el insurrance Policy esta relacionado con la combinacion de auto y dueno (ya que aunque un auto puede tener varios duenos y un dueno puede tener varios autos, la poliza es unica para cada combinacion) se agrega la columna Insurrance Policy en la tabla AutomobileOwners  con el insurrance ID.
6. Finalmente, la tabla Automobiles debe quedar con las columnas Car ID,VIN, Brand ID, Model ID, Year, Color ID, Insurrance ID e Insurrance Policy. Las otras tablas creadas serian Brands (Brand ID, Marca), Models (Model ID, Brand ID, Modelo), Colors (Color ID, Color), Owners (Owner ID, Owner Name, Owner Phone), Insurances (Insurrance ID, Insurrance Company) y AutomobileOwners (AutomobileOwner ID, Automobile ID, Owner ID).
